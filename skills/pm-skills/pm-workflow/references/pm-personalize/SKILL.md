# PM Personalize — 个人知识提炼

从项目 wiki 中识别跨项目通用的知识，提炼并写入个人知识库。

## 当由 pm-workflow 调度时

pm-personalize 不是工作流的标准阶段，而是按需触发：
- pm-knowledge 在 ingest 后可能建议用户提炼个人知识
- 用户在任意门控决策点可以要求执行个人知识提炼

### 触发方式

无需通过 Skill 工具显式调用。当 pm-knowledge 建议提炼或用户要求时，按此 skill 的流程执行。

### 执行后交接

完成后，向 pm-workflow 汇报：
- 提炼的知识条目列表
- 个人知识库新增路径

## 核心原则

- **项目知识归项目，通用知识归个人** — 个人库只存跨项目可复用的内容
- **草稿待审** — 所有提炼内容写入个人库时标记 `status: pending-review`，等用户确认
- **不覆盖** — 如果个人库已有相同主题，追加更新而非覆盖
- **引用溯源** — 每条提炼注明来自哪个项目 wiki

## 提炼标准

从项目 wiki 中筛选以下内容到个人库：

### 应该提炼（写入个人库）

| 类型 | 示例 | 写入路径 |
|------|------|---------|
| 技术约束/坑 | "这个项目踩到的坑，别的也可能遇到" | `insights/` |
| 通用方法论 | "总结出的工作流程、checklist" | `methods/` |
| 工具使用心得 | "某个工具的配置、踩坑、对比" | `tools/` |
| 可复用模板 | "PRD 模板、调研模板、决策模板" | `templates/` |
| 行业洞察 | "不依赖具体产品的市场规律、用户行为" | `industry/` |
| 能力模型 | "从项目中学到的技能、能力成长" | `skills/` |
| 可复用片段 | "通用的文案、流程描述、架构图说明" | `reusable/` |

### 不提炼（留在项目库）

- 具体产品功能描述
- 项目专属的竞品分析
- 项目特定的用户画像
- 项目决策记录（ADR）
- 带项目名/产品名的任何具体引用
- 项目时间线、里程碑

## 运行流程

### Step 1: 扫描项目 wiki

使用 Grep/Glob/Read 工具扫描项目 `.pm-wiki/` 目录：

1. 按修改时间排序，找出最近更新的页面（重点关注最近 7 天新增或大改的）
2. 检查 `decisions/` — 是否有通用决策模式
3. 检查 `constraints/` — 是否有可复用的约束认知
4. 检查 `synthesis/` — 是否有通用洞察
5. 检查 `requirements/` — 是否有通用需求模式
6. 检查 `competitors/` — 是否有跨产品通用的竞品规律

### Step 2: 识别可提炼内容

对每个候选页面，判断：
- **去掉项目名后，内容是否依然有意义？** → 是则提炼
- **其他项目是否可能遇到相同问题？** → 是则提炼
- **这是经验/规律，还是具体事实？** → 经验/规律才提炼

### Step 3: 提炼并写入个人库

对每个识别到的提炼项：

1. 检查个人库 `~/.pm-wiki/` 是否已有类似主题
   - 已有 → 追加更新，标注更新来源
   - 无 → 创建新页面

2. 页面格式：
```markdown
---
type: analysis | method | template | reference
status: pending-review
source: <项目 wiki 路径>
ingested: YYYY-MM-DD
tags: [提炼的标签]
---
# 主题

> [待审] 此内容由 pm-personalize 从 [项目名] 提炼，请审核后确认

[提炼后的内容，已去除项目特异性引用]

## 来源
- [项目知识库] <project>/.pm-wiki/<分类>/<页面>.md
```

3. 在个人库的 `log.md` 中记录本次提炼

### Step 4: 向用户报告

运行结束后，向用户展示：

```
## 知识提炼报告

### 扫描范围
- 项目：[项目名]
- 扫描页面：X 个
- 最近修改：N 个

### 提炼结果
| 状态 | 个人库路径 | 来源 |
|------|-----------|------|
| [待审] | insights/xxx.md | decisions/xxx.md |
| [待审] | methods/xxx.md | synthesis/xxx.md |
| 跳过 | — | users/xxx.md (项目特定) |

### 待您确认
以上标记 [待审] 的页面，请确认后告知，我将移除待审标签。
```

## 触发方式

1. **手动触发** — 用户调用 `/pm-personalize` 或说"提炼个人知识"
2. **定时触发** — 通过 Claude Code 的 CronCreate 工具设置每日/每周运行
3. **主动触发** — 在 pm-knowledge 完成 ingest 后，如果发现新内容较多（超过 3 个页面更新），主动询问用户是否运行 pm-personalize
